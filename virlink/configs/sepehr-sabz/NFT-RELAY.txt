نقش‌ها در virlink
-----------------
  client  = IRAN   (95.38.195.35)
  server  = KHAREJ (64.118.156.193)

نام‌گذاری [mangle] در config
----------------------------
  IRAN (client/tcp.toml):
    srcip = SRC_IRAN   = 185.41.1.52
    dstip = WIRE_KHAREJ = 37.152.181.38
    remote_ip = REAL_KHAREJ = 64.118.156.193

  KHAREJ (server/tcp.toml):
    srcip = SRC_KHAREJ  = 37.152.181.38
    dstip = WIRE_IRAN   = 185.41.1.52
    remote_ip = REAL_IRAN = 95.38.195.35

فلسفه (مثل GRE — input/output hooks)
--------------------------------------
  stack از IP واقعی connect/listen می‌کند
  nft روی wire هدر را عوض می‌کند:

  IRAN  TX output:  daddr REAL_KHAREJ → saddr SRC_IRAN
  IRAN  RX input:   saddr WIRE_KHAREJ → REAL_KHAREJ (stack)

  KHAREJ TX output: daddr WIRE_IRAN → saddr SRC_KHAREJ
  KHAREJ RX input:  saddr WIRE_IRAN → REAL_IRAN (stack)

مثال TCP SYN
------------
  IRAN stack: connect 95.38.195.35 → 64.118.156.193:8443
  wire TX:    185.41.1.52 → 64.118.156.193  (nft output)

  KHAREJ wire RX: 185.41.1.52 → 64.118.156.193:8443
  KHAREJ stack: accept از REAL_IRAN (nft input)

  KHAREJ wire TX: 37.152.181.38 → 95.38.195.35  (nft output)
  IRAN wire RX:   37.152.181.38 → 95.38.195.35
  IRAN stack:     socket می‌بیند REAL_KHAREJ (nft input)

فایروال KHAREJ
----------------
  TCP 8443 از WIRE_IRAN (185.41.1.52) — نه فقط REAL_IRAN

بررسی
-----
  nft list table ip virlink_mangle
  startup log: [wire] nft relay loaded  counters in=… out=…

  tcpdump KHAREJ: syn از 185.41.1.52 باید بیاید

تunnel types با wire spoof ([mangle])
-------------------------------------
  icmp, udp, bip, tcp, tcpmux,
  gre, gre-fou, ipip-fou, bonded-gre-fou, l2tpv3, gre-fou-ipsec

  TCP و tcpmux همان مدل nft relay بالا را استفاده می‌کنند (kernel socket +
  input/output mangling). تفاوت فقط در framing روی stream است؛ tcpmux flow-hash
  multiplexing دارد (configs: client/tcpmux.toml, server/tcpmux.toml, port 8444).
